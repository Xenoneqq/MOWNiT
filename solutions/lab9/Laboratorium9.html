Zadanie 1

Celem zadania jest przeksztacenie podanych rwna rniczkowych wyszych rzdw do ukadw rwna rniczkowych pierwszego rzdu. Tego typu operacja jest istotna zarwno w analizie teoretycznej, jak i w obliczeniach numerycznych wiele algorytmw (np. metoda Rungego-Kutty) dziaa jedynie na ukadach rwna pierwszego rzdu.

(a) Rwnanie Van der Pola

Dane jest nieliniowe rwnanie rniczkowe drugiego rzdu:

math
y'' = y'(1 - y^2) - y

Aby sprowadzi to rwnanie do postaci ukadu pierwszego rzdu, wprowadzamy nastpujce zmienne pomocnicze:

math
y_1 = y, \quad y_2 = y'

Wwczas otrzymujemy ukad:

math
\begin{cases}
y_1' = y_2 \\
y_2' = y_2(1 - y_1^2) - y_1
\end{cases}

(b) Rwnanie Blasiusa

Dane jest nieliniowe rwnanie trzeciego rzdu:

math
y''' = -y \cdot y''

Wprowadzamy nowe zmienne:

math
y_1 = y, \quad y_2 = y', \quad y_3 = y''

Otrzymujemy ukad:

math
\begin{cases}
y_1' = y_2 \\
y_2' = y_3 \\
y_3' = -y_1 \cdot y_3
\end{cases}

Takie podejcie pozwala na atwe rozwizanie numeryczne tego typu rwna, np. przy pomocy scipy.integrate.solve_ivp.

(c) II zasada dynamiki Newtona dla problemu dwch cia

Rozwaamy rwnania opisujce ruch ciaa pod wpywem grawitacji w dwch wymiarach:

math
\begin{aligned}
y_1'' &= -\frac{GM \cdot y_1}{(y_1^2 + y_2^2)^{3/2}} \\
y_2'' &= -\frac{GM \cdot y_2}{(y_1^2 + y_2^2)^{3/2}}
\end{aligned}

S to rwnania drugiego rzdu, ktre przeksztacamy do postaci pierwszego rzdu, wprowadzajc zmienne:

math
y_3 = y_1', \quad y_4 = y_2'

Wwczas cay ukad przyjmuje posta:

math
\begin{cases}
y_1' = y_3 \\
y_2' = y_4 \\
y_3' = -\frac{GM \cdot y_1}{(y_1^2 + y_2^2)^{3/2}} \\
y_4' = -\frac{GM \cdot y_2}{(y_1^2 + y_2^2)^{3/2}}
\end{cases}

Taki ukad opisuje pen dynamik ruchu punktu materialnego w centralnym polu grawitacyjnym i moe by bezporednio uyty do analizy orbitalnej.

Zadanie 2

Przeksztacamy dany problem pocztkowy do postaci autonomicznej (tzn. bez jawnej zalenoci od zmiennej ( t ) w prawej stronie rwna).

Dany ukad:

math
\begin{aligned}
y_1' &= \frac{y_1}{t} + y_2 t \\
y_2' &= \frac{t(y_2^2 - 1)}{y_1} \\
y_1(1) &= 1 \\
y_2(1) &= 0
\end{aligned}

Przeksztacenie do postaci autonomicznej

Wprowadzamy now zmienn:

math
y_3 = t

i dodajemy nowe rwnanie opisujce jej ewolucj:

math
y_3' = 1

Podstawiamy $t = y_3$ do pozostaych rwna:

math
\begin{aligned}
y_1' &= \frac{y_1}{y_3} + y_2 y_3 \\
y_2' &= \frac{y_3 (y_2^2 - 1)}{y_1} \\
y_3' &= 1
\end{aligned}

Warunki pocztkowe

Z oryginalnych warunkw:

math
y_1(1) = 1, \quad y_2(1) = 0

Wynika:

math
y_3(1) = 1

Zatem ostateczny autonomiczny problem pocztkowy ma posta:

math
\begin{cases}
y_1' = \dfrac{y_1}{y_3} + y_2 y_3 \\
y_2' = \dfrac{y_3 (y_2^2 - 1)}{y_1} \\
y_3' = 1 \\
y_1(1) = 1, \quad y_2(1) = 0, \quad y_3(1) = 1
\end{cases}

Podsumowanie

Dziki wprowadzeniu zmiennej $y_3 = t$, udao si sprowadzi nieautonomiczny ukad do postaci autonomicznej, co umoliwia jego pniejsz analiz numeryczn lub teoretyczn z uyciem metod dla autonomicznych ukadw ODE.

Zadanie 3

Rozwamy problem pocztkowy:

math
\begin{cases}
y' = \sqrt{1 - y} \\
y(0) = 0
\end{cases}

Zbadamy, czy funkcja:

math
y(t) = \dfrac{t(4 - t)}{4}

jest rozwizaniem tego problemu oraz wyznaczymy dziedzin, na ktrej to rozwizanie jest poprawne.

1. Sprawdzenie warunku pocztkowego

Podstawiamy $t = 0$:

math
y(0) = \frac{0 \cdot (4 - 0)}{4} = 0

Zatem funkcja spenia warunek pocztkowy.

2. Obliczenie pochodnej $y'(t)$

Rozwijamy wzr:

math
y(t) = \frac{4t - t^2}{4}

Liczymy pochodn:

math
y'(t) = \frac{4 - 2t}{4}

3. Sprawdzenie rwnania rniczkowego

Rwnanie:

math
y'(t) = \sqrt{1 - y(t)}

Lewa strona:

math
y'(t) = \frac{4 - 2t}{4}

Prawa strona:

math
\sqrt{1 - y(t)} = \sqrt{1 - \frac{t(4 - t)}{4}} = \sqrt{\frac{4 - t(4 - t)}{4}}

Rozwijamy licznik:

math
t(4 - t) = 4t - t^2 \Rightarrow 4 - (4t - t^2) = 4 - 4t + t^2

Zatem:

math
\sqrt{1 - y(t)} = \sqrt{ \frac{t^2 - 4t + 4}{4} } = \sqrt{ \frac{(t - 2)^2}{4} } = \frac{|t - 2|}{2}

Porwnujemy obie strony:

math
y'(t) = \frac{4 - 2t}{4} = \frac{2 - t}{2}

Zatem:

math
y'(t) = \frac{2 - t}{2}, \quad \sqrt{1 - y(t)} = \frac{|t - 2|}{2}

Aby rwnanie byo spenione:

math
\frac{2 - t}{2} = \frac{|t - 2|}{2}

To prawda tylko wtedy, gdy:

math
2 - t = |t - 2| \Rightarrow t \leq 2

Bo:

  • dla $t < 2$, $|t - 2| = 2 - t$ zgadza si,
  • dla $t > 2$, $|t - 2| = t - 2$, wic rwnanie nie jest spenione.

4. Dziedzina rozwizania

Funkcja spenia rwnanie rniczkowe tylko wtedy, gdy:

math
t \leq 2

Dodatkowo musimy uwzgldni dziedzin pierwiastka:

math
\sqrt{1 - y(t)} \Rightarrow 1 - y(t) \geq 0

Zatem:

math
y(t) \leq 1

Sprawdzamy, dla jakich $t$ zachodzi $y(t) \leq 1$:

math
y(t) = \frac{t(4 - t)}{4} \leq 1

Pomnmy obie strony przez 4:

math
t(4 - t) \leq 4 \Rightarrow 4t - t^2 \leq 4
\Rightarrow -t^2 + 4t - 4 \leq 0

To jest nierwno kwadratowa. Sprawdmy pierwiastki:

math
-t^2 + 4t - 4 = 0 \Rightarrow t^2 - 4t + 4 = 0 \Rightarrow (t - 2)^2 = 0 \Rightarrow t = 2

Zatem nierwno speniona jest dla:

math
t \leq 2

Ostateczna dziedzina

Funkcja $y(t) = \frac{t(4 - t)}{4}$ spenia rwnanie i warunek pocztkowy dla $t \in [0, 2]$.

Podsumowanie

Funkcja:

math
y(t) = \frac{t(4 - t)}{4}

spenia rwnanie rniczkowe $y' = \sqrt{1 - y}$ oraz warunek pocztkowy $y(0) = 0$. Dziedzin, dla ktrej stanowi rozwizanie problemu pocztkowego, jest:

math
t \in [0, 2]

Zadanie 4

Rozwaamy rwnanie rniczkowe:

math
y' = -5y, \quad y(0) = 1

Zbadamy jego stabilno oraz zastosujemy metody numeryczne (jawna i niejawna metoda Eulera) do wyznaczenia rozwizania przy rnych krokach.

(a) Analityczna stabilno

Rwnanie ma rozwizanie analityczne:

math
y(t) = e^{-5t}

Rozwizanie zbiega do 0 przy $t \to \infty$, wic ukad jest stabilny analitycznie ma charakter tumiony.

(b) Zbieno metody Eulera porwnanie numeryczne i analityczne

Sprawdzimy zbieno metody Eulera dla rnych wartoci kroku ( h ), porwnujc wynik numeryczny z dokadnym rozwizaniem.

In[20]:
import numpy as np
import matplotlib.pyplot as plt

# Dokadne rozwizanie
def y_exact(t):
    return np.exp(-5 * t)

# Metoda Eulera (jawna)
def euler_explicit(f, y0, h, t_end):
    t_values = np.arange(0, t_end + h, h)
    y_values = np.zeros_like(t_values)
    y_values[0] = y0
    for i in range(1, len(t_values)):
        y_values[i] = y_values[i-1] + h * f(t_values[i-1], y_values[i-1])
    return t_values, y_values

# Prawa strona rwnania
f = lambda t, y: -5 * y

# Porwnanie dla rnych krokw
h_vals = [0.5, 0.25, 0.1]
t_max = 2.0
t_plot = np.linspace(0, t_max, 200)

plt.figure(figsize=(10,6))
plt.plot(t_plot, y_exact(t_plot), label="Dokadne rozwizanie", color="black")

for h in h_vals:
    t_num, y_num = euler_explicit(f, y0=1, h=h, t_end=t_max)
    plt.plot(t_num, y_num, 'o-', label=f"Euler jawny (h={h})")

plt.title("Porwnanie: metoda Eulera vs rozwizanie dokadne")
plt.xlabel("t")
plt.ylabel("y(t)")
plt.legend()
plt.grid(True)
plt.show()

Jak wida, dla mniejszych krokw ( h ) metoda Eulera zblia si do rozwizania dokadnego, co potwierdza jej zbieno.

(c) Stabilno numeryczna metody Eulera dla h = 0.5

Sprawdzamy, czy metoda Eulera jawna jest stabilna przy kroku ( h = 0.5 ). Dla rwnania ( y' = \lambda y ), stabilno jawnej metody wymaga:

math

Podstawiajc $\lambda = -5$, mamy:

math
|1 - 5 \cdot 0.5| = 1.5 > 1

Metoda nie jest stabilna dla tego kroku.

(e) Stabilno metody Eulera niejawnej

(h) Iteracja bezporednia warunek zbienoci

Dla iteracji:

math
y_{n+1}^{(k+1)} = y_n - 5h y_{n+1}^{(k)}

funkcja iteracyjna to $\varphi(y) = y_n - 5h y$, ktrej pochodna to $\varphi'(y) = -5h$. Dla zbienoci iteracji wymagamy:

math
|\varphi'(y)| = 5h < 1 \Rightarrow h < 0.2

Zatem metoda iteracyjna dziaa poprawnie dla $h < 0.2$.

Zadanie 5

Rozwaamy ukad rwna:

math
\begin{cases}
y_1' = -2 y_1 + y_2, \\
y_2' = - y_1 - 2 y_2.
\end{cases}

Stosujc metod Eulera jawn do tego ukadu, iterujemy wedug:

$$ \mathbf{y}_{n+1} = (I + hA) \mathbf{y}_n, $$

gdzie macierz $A$ opisuje ukad:

$$ A = \begin{bmatrix} -2 & 1 \\ -1 & -2 \end{bmatrix}. $$

Stabilno numeryczna metody Eulera wymaga, by wszystkie wartoci wasne macierzy iteracyjnej $M = I + hA$ miay modu mniejszy lub rwny 1.

Kod w Python i wyjanienie

In[24]:
import numpy as np

# Macierz A opisujca ukad
A = np.array([[-2, 1],
              [-1, -2]])

# Funkcja oblicza maksymalny modu wartoci wasnych macierzy M = I + hA
def max_eigenvalue_mod(h):
    M = np.eye(2) + h * A
    eigvals = np.linalg.eigvals(M)
    return np.max(np.abs(eigvals))

# Przeszukujemy wartoci kroku h od 0 do 1, aby znale maksymalny stabilny krok
h_values = np.linspace(0, 1, 1000)
stable_h = [h for h in h_values if max_eigenvalue_mod(h) <= 1]

print(f"Maksymalny stabilny krok h = {max(stable_h):.3f}")

Maksymalny stabilny krok h = 0.800

Co robi ten kod?

  • Definiuje macierz $A$ opisujc ukad rwna.
  • Dla kadej wartoci kroku $h$ z zakresu [0, 1] tworzy macierz iteracyjn $M = I + hA$.
  • Oblicza wartoci wasne $M$ i wybiera ich maksymalny modu.
  • Sprawdza, dla ktrych $h$ maksymalny modu wartoci wasnych nie przekracza 1.
  • Wywietla najwikszy krok $h$ speniajcy warunek stabilnoci.

Wniosek

Z analizy i symulacji wynika, e metoda Eulera jawna dla tego ukadu jest stabilna dla krokw $h \leq 0{,}8$.

Zadanie 6

Rozpatrujemy rwnanie rniczkowe:

math
y' = \alpha t^{\alpha - 1}, \quad y(0) = 0,

ktrego dokadnym rozwizaniem jest

math
y(t) = t^{\alpha}.

Przebieg oblicze i ich interpretacja

W zadaniu numerycznie rozwizalimy to rwnanie metod Eulera dla trzech wartoci parametru $\alpha$: 2.5, 1.5 oraz 1.1. Dla kadej z nich zastosowalimy trzy rne kroki czasowe $h = 0.2, 0.1, 0.05$.

Implementacja polega na dyskretnym podejciu do rozwizania rwnania, gdzie w kolejnych punktach czasowych wyliczamy przyblion warto funkcji $y$ na podstawie wartoci pochodnej w poprzednim punkcie. Oznacza to, e posuwamy si krok po kroku, korzystajc z rwnania rnicowego, aby oszacowa kolejne wartoci.

Nastpnie, dla kadego kroku i kadej wartoci $\alpha$, porwnujemy warto otrzyman metod numeryczn z wartoci dokadn $t^\alpha$ w punkcie $t=1$. Pozwala to oceni bd metody, ktry jest miar jej dokadnoci.

Aby okreli, jak szybko metoda staje si dokadniejsza przy zmniejszaniu kroku, obliczamy empiryczny rzd zbienoci. Jest to liczba opisujca, jak bardzo bd zmniejsza si wraz ze zmniejszeniem kroku $h$. Teoretycznie metoda Eulera jest pierwszego rzdu, wic oczekujemy, e bd bdzie proporcjonalny do $h$.

Kod numeryczny

import numpy as np

def euler_method(alpha, h):
    t = np.arange(0, 1 + h, h)
    y = np.zeros_like(t)
    for i in range(len(t) - 1):
        f = alpha * t[i]**(alpha - 1)
        y[i+1] = y[i] + h * f
    return t, y

def exact_solution(alpha, t):
    return t**alpha

def compute_error(y_num, y_exact):
    return np.abs(y_num - y_exact)

def empirical_order(error1, error2, h1, h2):
    return np.log(error1 / error2) / np.log(h1 / h2)

alphas = [2.5, 1.5, 1.1]
hs = [0.2, 0.1, 0.05]

for alpha in alphas:
    errors = []
    print(f"\nAlpha = {alpha}")
    for h in hs:
        t, y_num = euler_method(alpha, h)
        y_ex = exact_solution(alpha, t)
        error = compute_error(y_num, y_ex)[-1]
        errors.append(error)
        print(f"h = {h:.2f}, Bd = {error:.5e}")

    for i in range(len(hs) - 1):
        p = empirical_order(errors[i], errors[i+1], hs[i], hs[i+1])
        print(f"Empiryczny rzd zbienoci midzy h={hs[i]:.2f} a h={hs[i+1]:.2f}: p  {p:.3f}")

Wnioski z przeprowadzonych oblicze

Zmniejszanie kroku numerycznego prowadzi do systematycznego spadku bdu, co potwierdza poprawno implementacji oraz typowe wasnoci metody Eulera. Empiryczny rzd zbienoci jest bliski 1, zgodnie z teori, co oznacza liniow zbieno metody wzgldem kroku $h$.

Dla $\alpha$ bliskiego 1 obserwujemy nieco niszy rzd zbienoci, co mona tumaczy mniejsz gadkoci funkcji pochodnej w pobliu zera. Dla wyszych wartoci $\alpha$ metoda zachowuje si zgodnie z oczekiwaniami, osigajc typowy pierwszy rzd zbienoci.

Podsumowujc, metoda Eulera jest efektywna do numerycznego rozwizywania tego problemu, a wyniki numeryczne potwierdzaj jej waciwoci teoretyczne.
